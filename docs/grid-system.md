🛠 Технічне завдання: Модульний конструктор карти магазину (Web)
1. Концепція системи
Система являє собою інтерактивну сітку (Grid), де адміністратор може створювати об'єкти (стелажі, холодильники, каси), задавати їм розміри та параметри (кількість полиць). Кожен об'єкт прив'язаний до координат сітки.

2. Технологічний стек
Frontend: React.js (або Vue.js) — для інтерфейсу.

Двигун карти: Konva.js або Fabric.js. Ці бібліотеки дозволяють працювати з Canvas, реалізувати Drag-and-drop, зміну розмірів об'єктів та зум (масштабування).

Локальне збереження: IndexedDB (через бібліотеку Dexie.js) або LocalStorage. Це дозволить працювати без сервера на першому етапі.

Експорт: Можливість збереження карти в JSON-файл.

3. Функціональні вимоги до конструктора
А. Налаштування сітки (Grid)
Адмін задає розмір магазину (наприклад, 20x30 метрів).

Крок сітки (наприклад, 1 квадрат = 0.5 метра).

Функція "Прив'язка до сітки" (Snapping), щоб об'єкти ставали рівно.

Б. Робота з об'єктами
Створення: Вибір типу об'єкта з панелі (Стелаж, Холодильник, Вітрина).

Редагування:

Зміна назви (наприклад, "Холодильник Молочка 1").

Вказання кількості полиць (Shelves).

Зміна кольору або іконки.

Видалення/Копіювання.

В. Ієрархія даних (Важливо для БД)
Об'єкт має зберігатися у форматі JSON з такою структурою:

JSON
{
  "id": "unique_id_123",
  "name": "Стелаж Бакалія",
  "type": "shelf",
  "position": { "x": 10, "y": 5 },
  "size": { "width": 2, "height": 1 }, // в юнітах сітки
  "shelves": [
    { "level": 1, "status": "ok" },
    { "level": 2, "status": "warning" },
    { "level": 3, "status": "critical" }
  ]
}
4. Візуалізація та статуси
Кожна полиця всередині об'єкта може мати динамічний статус, який змінює колір об'єкта на карті:

Зелений: Всі товари свіжі.

Жовтий: Є товари, термін яких виходить (наприклад, через 3 дні).

Червоний: Протермінування або критичний термін (сьогодні/завтра).

5. Логіка зберігання даних (Локально)
Для MVP ми не використовуємо сервер. Використовуємо Dexie.js (зручна обгортка над IndexedDB):

При кожній зміні (перетягування, зміна назви) стан карти зберігається в браузері.

Навіть після перезавантаження сторінки карта залишається.

Кнопка "Завантажити JSON" — дозволяє зберегти макет магазину як файл на комп'ютер.
